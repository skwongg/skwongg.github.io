    Arrays and Hashes

    My current understanding.

    January 30, 2015

    Array = items referenced numerically [0â€¦n] Hash = items referenced by keys

    Arrays and hashes have similar functionality in that they can store variables and allow for each variable to be accessed. It is the way in which the stored variables can be accessed that sets them apart. Arrays store variables in a numerical index starting from 0. The first element/variable in an array is at position 0.

    So for an array named Ex_Array, Ex_Array[0] would be the first. The syntax to define an array and the first element could be:

    Ex_Array = [a, b, c] Ex_Array[0] = a Ex_Array[1] = b Ex_Array[2] = c

    Ex_Hash = { "0" => a, "1" => b, "2" => c } #mirrors array Ex_Hash = { "x" => elephant, "asdf" => donkey, "coffee"=> love}

    In both examples above with calling the 0th position on Ex_Array or calling Ex_Hash with a "0" key, they would return "a". So what are the differences between the two, and which types of situations call for which?

    From research on the internet, both methods can index things and get the job done, but as far as speed, efficiency and complexity, they can be very different. In an array, things are combed through in order based on index (0,1,2,3...etc.) whereas hashes are like subsets of indexes.

    A similar example would be in describing the order of words in a dictionary. Imagine a dictionary that gave you an ordered list of words with their associated definitions and the only way you could access the word you needed was by going through each word individually in order until you arrived at the word you wanted.

    A hash in the dictionary example would be that each hash key is represented by a letter in the dictionary (a,b,c...z). If you wanted to access words that start with the letter "A" you could do so via hash. In the dictionary example, each letter in the alphabet serves as a key to a specific hash, and within each hash is the indexed information (words that start with a particular letter).

    In real world examples, one of the biggest advantages (based on my understanding) of hashes or the situations in which hashes can shine are in complex/gigantic data structures. There are situations in which organizing data based in several hashes may serve to be substantially faster than iterating through each item of an array. It is generally accepted that arrays are simpler to set up and equally functional in most instances to access data, however the differences in complexity/function are where hashes can shine.
